# GUIDE-00. 시작하기 전에 — 이 가이드를 읽는 방법

> 이 프로젝트는 **비개발자 출신 주니어**가 전자정부프레임워크의 구조를
> 처음부터 하나씩 이해할 수 있도록 설계된 학습 프로젝트입니다.

---

## 이 프로젝트로 무엇을 배우나요?

한국 공공기관 프로젝트의 95% 이상은 **전자정부프레임워크** 기반입니다.
이 프레임워크는 다음 4개 계층으로 구성됩니다:

```
요청(클릭/입력)
    ↓
Controller  ← "안내 데스크" — 요청을 받아서 담당자에게 넘겨줌
    ↓
Service     ← "실무 담당자" — 실제 일을 처리함
    ↓
DAO         ← "DB 창구" — 데이터베이스에서 데이터를 꺼내거나 저장함
    ↓
DB (H2)     ← "서랍장" — 데이터가 실제로 저장되는 곳
    ↑
VO          ← "상자" — 각 계층 사이에서 데이터를 담아 전달하는 그릇
```

## 프로젝트 실행 전 준비 — 환경 설정

> 처음 시작이라면 이 단계를 반드시 따라해야 합니다.
> 이미 설치된 분은 "설치 확인" 부분만 확인하세요.

---

### STEP 0. 터미널(명령 프롬프트) 여는 방법

코드를 실행하려면 "터미널"이라는 검은 창을 사용해야 합니다.
처음에는 낯설지만, 결국 명령어를 입력하는 창입니다.

**Windows (명령 프롬프트 열기):**
1. 키보드에서 `Windows 키 + R` 동시에 누르기
2. 실행 창에 `cmd` 입력 후 Enter
3. 검은 창이 뜨면 성공

또는:
1. 시작 메뉴 클릭
2. "명령 프롬프트" 검색
3. 클릭해서 실행

**Mac (터미널 열기):**
1. `Command(⌘) + Space` 동시에 누르기 (스포트라이트 검색)
2. "터미널" 입력 후 Enter
3. 흰색 또는 검은 창이 뜨면 성공

---

### STEP 1. Java 설치 확인

방금 연 터미널 창에 아래 명령어를 입력하고 Enter를 누르세요:

```bash
java -version
```

**성공한 경우 (이런 메시지가 나오면 OK):**
```
java version "11.0.xx" 2024-xx-xx
Java(TM) SE Runtime Environment ...
```
→ 숫자가 11 이상이면 됩니다. 17, 21이어도 OK.

**실패한 경우 (이런 메시지가 나오면 Java가 없는 것):**
```
'java'은(는) 내부 또는 외부 명령이 아닙니다
```
또는
```
command not found: java
```

**Java가 없다면 → 설치해야 합니다:**

1. 웹 브라우저에서 검색: `Adoptium JDK 11 다운로드`
2. https://adoptium.net 접속
3. `Temurin 11 (LTS)` 선택 → 본인 운영체제에 맞는 설치파일 다운로드
   - Windows: `.msi` 파일
   - Mac: `.pkg` 파일
4. 다운로드한 파일 더블클릭 → "다음" 계속 클릭하여 설치
5. 설치 완료 후 터미널을 **새로 열고** `java -version` 다시 실행

---

### STEP 2. Maven 설치 확인

터미널에 입력:

```bash
mvn -version
```

**성공한 경우:**
```
Apache Maven 3.x.x ...
```

**실패한 경우 (Maven이 없는 것):**
```
'mvn'은(는) 내부 또는 외부 명령이 아닙니다
```

**Maven이 없다면 → 설치해야 합니다:**

1. 웹 브라우저에서 검색: `Apache Maven 다운로드`
2. https://maven.apache.org/download.cgi 접속
3. `Binary zip archive` 다운로드 (예: apache-maven-3.9.x-bin.zip)
4. 압축 해제 후 원하는 폴더에 복사 (예: `C:\maven` 또는 `/opt/maven`)
5. 환경변수 설정 (이 부분이 복잡할 수 있습니다):

   **Windows:**
   - 시작 메뉴 → "환경 변수" 검색 → "시스템 환경 변수 편집"
   - `환경 변수` 버튼 클릭
   - 시스템 변수에서 `Path` 찾아 편집
   - 새로 만들기 → Maven을 압축 푼 경로 + `\bin` 추가
     (예: `C:\maven\apache-maven-3.9.x\bin`)
   - 확인 → 터미널 새로 열고 `mvn -version` 확인

   **Mac:**
   ```bash
   # 터미널에 입력 (압축 푼 경로에 맞게 수정)
   export PATH=/opt/maven/apache-maven-3.9.x/bin:$PATH
   ```

> 💡 **팁:** Maven 설치가 복잡하면 구글에서 "Mac Maven 설치" 또는 "Windows Maven 설치"를 검색하면 스크린샷이 있는 상세 가이드를 찾을 수 있습니다.

---

### STEP 3. IntelliJ IDEA 설치 (권장 IDE)

IDE(통합 개발 환경)는 코드를 읽고 편집하는 전용 프로그램입니다.
메모장으로도 볼 수 있지만, IDE를 쓰면 코드 색깔, 자동완성, 파일 탐색이 훨씬 편합니다.

**IntelliJ IDEA Community Edition (무료):**

1. 검색: `IntelliJ IDEA 다운로드`
2. https://www.jetbrains.com/idea/download 접속
3. `Community` 탭 선택 (무료 버전) → 다운로드
4. 설치파일 실행 → 기본 설정으로 설치

**프로젝트 열기 (IntelliJ):**
1. IntelliJ 실행
2. `Open` 클릭
3. 이 프로젝트 폴더(`egov`) 선택 → `OK`
4. 오른쪽 아래에서 Maven 인덱싱이 완료될 때까지 기다림 (몇 분 소요)

> 💡 Eclipse를 사용한다면: `File → Import → Maven → Existing Maven Projects` 선택 후 폴더 지정

### STEP 4. 프로젝트 폴더 위치 확인

이 가이드에서 계속 파일을 "열어보세요"라고 합니다.
파일이 어디 있는지 알아야 열 수 있습니다.

**폴더 구조 (예시, 실제 위치는 다를 수 있음):**
```
내 컴퓨터
└── 사용자 폴더 (예: C:\Users\홍길동\ 또는 /Users/홍길동/)
    └── workspace/  (또는 다운로드/ 등 저장한 곳)
        └── egov/                    ← 이 프로젝트의 최상위 폴더
            ├── pom.xml
            ├── docs/
            │   └── guide/           ← 지금 읽고 있는 가이드 파일들
            └── src/
                └── main/
                    ├── java/        ← Java 소스코드
                    ├── resources/   ← 설정 파일
                    └── webapp/      ← JSP 화면 파일
```

**파일 탐색기/파인더에서 찾기:**
- Windows: 탐색기(폴더 아이콘)에서 위 경로 탐색
- Mac: 파인더에서 `Command + Shift + G` → 경로 직접 입력

**IntelliJ에서 파일 찾기:**
- 왼쪽 `Project` 패널에서 폴더 트리 탐색
- `Ctrl + Shift + N` (Windows) 또는 `Cmd + Shift + O` (Mac): 파일 이름으로 검색

## 프로젝트 실행 방법

환경 설정이 완료되었다면 이제 실행해봅시다.

### 방법 A. 터미널에서 실행 (권장)

**1단계:** 터미널을 열고 프로젝트 폴더로 이동합니다.

```bash
# 예시: 프로젝트가 바탕화면에 있을 경우
cd Desktop/egov

# 예시: 프로젝트가 /Users/홍길동/workspace/egov 에 있을 경우 (Mac)
cd /Users/홍길동/workspace/egov

# 예시: 프로젝트가 C:\workspace\egov 에 있을 경우 (Windows)
cd C:\workspace\egov
```

> `cd`는 "Change Directory"의 약자로, 폴더를 이동하는 명령어입니다.

**2단계:** 제대로 이동했는지 확인합니다.

```bash
# Mac/Linux
ls

# Windows
dir
```

`pom.xml` 파일이 목록에 보이면 올바른 위치입니다.

**3단계:** 서버를 실행합니다.

```bash
mvn spring-boot:run
```

처음 실행 시 필요한 파일을 인터넷에서 다운받기 때문에 **5~10분** 정도 걸릴 수 있습니다.
(인터넷이 연결되어 있어야 합니다)

**성공했을 때 나오는 메시지:**
```
  .   ____          _            __ _ _
 /\\ / ___'_ __ _ _(_)_ __  __ _ \ \ \ \
...
Started EgovApplication in 3.xxx seconds (JVM running for 4.xxx)
```

이 메시지가 나오면 서버가 켜진 것입니다!

### 방법 B. IntelliJ에서 실행

1. IntelliJ에서 프로젝트를 열고
2. 왼쪽 프로젝트 탐색기에서 `src/main/java/.../EgovApplication.java` 파일 찾기
3. 파일을 열고 초록색 ▶ 버튼 클릭
4. 또는 파일 위에서 마우스 오른쪽 버튼 → `Run 'EgovApplication'` 선택

---

### STEP 5. 브라우저에서 확인

서버가 켜진 상태에서:

1. Chrome, Edge, Safari 등 웹 브라우저 열기
2. 주소창에 입력: `http://localhost:8080/board/list`
3. Enter

**게시판 목록이 보이면 완료입니다!**

**안 보이는 경우 체크리스트:**
- [ ] 터미널 창에 에러 메시지가 없는지 확인 (빨간 글씨 ERROR)
- [ ] 서버가 실행 중인 터미널 창을 닫지 않았는지 확인
- [ ] 주소가 정확한지 확인 (`localhost:8080`, 대소문자 주의)
- [ ] 8080 포트를 다른 프로그램이 쓰고 있으면 충돌 가능

## 공부할 때 도움이 되는 팁

**1. 겁먹지 마세요**
- 처음에는 모든 코드를 이해하려고 하지 않아도 됩니다
- 각 파일이 "무슨 역할"을 하는지만 먼저 파악하세요

**2. 순서가 중요합니다**
- VO → DAO → Service → Controller → JSP 순서로 읽으세요
- 아래 계층(VO)을 먼저 이해해야 위 계층(Controller)이 보입니다

**3. 실제로 실행해보세요**
- 코드를 보기만 하지 말고, 실행해서 화면을 보세요
- 코드를 살짝 바꿔보고 어떻게 달라지는지 확인하세요
- 망가뜨려도 괜찮습니다 — 원본 파일을 백업해두거나, git으로 되돌릴 수 있어요

**4. 에러가 나면 당황하지 말고**
- 터미널 창에서 `ERROR` 또는 `Exception` 글자가 적힌 줄을 찾으세요
- 그 줄의 내용을 구글에 복사해서 검색해보세요
- 대부분의 에러는 이미 누군가 해결한 내용이 인터넷에 있습니다

**5. 로그를 보는 습관**
- 실행 중 터미널 창에 찍히는 로그를 자주 보세요
- 에러가 나면 로그에 이유가 나옵니다

## 자주 나오는 용어 정리

| 용어 | 쉬운 설명 |
|------|---------|
| **프레임워크** | 개발할 때 쓰는 "틀". 기본 뼈대를 미리 만들어 둔 것 |
| **Spring** | 자바 웹 개발에 가장 많이 쓰이는 프레임워크 |
| **Maven** | 프로젝트에 필요한 라이브러리를 자동으로 다운받아주는 도구 |
| **MyBatis** | Java 코드와 SQL을 연결해주는 도구 |
| **H2** | 별도 설치 없이 쓸 수 있는 학습용 DB |
| **JSP** | HTML + Java를 같이 쓸 수 있는 화면 파일 (.jsp) |
| **JSTL** | JSP에서 if문/반복문을 HTML처럼 쓸 수 있게 해주는 태그 라이브러리 |
| **VO** | Value Object. 데이터를 담는 상자(객체) |
| **DAO** | Data Access Object. DB와 직접 소통하는 코드 |
| **Controller** | 브라우저 요청을 처음으로 받는 코드 |
| **Service** | 실제 비즈니스 로직(업무 처리)을 담당하는 코드 |
| **어노테이션** | `@`로 시작하는 코드. Spring에게 "이 클래스는 ○○입니다"라고 알려주는 표시 |
| **IDE** | IntelliJ, Eclipse 같은 코드 편집 전용 프로그램 |
| **터미널** | 명령어를 입력해서 컴퓨터를 조작하는 검은 창 (Windows: cmd, Mac: 터미널) |
| **localhost** | 내 컴퓨터를 가리키는 주소. 서버를 내 컴퓨터에서 실행할 때 사용 |
| **8080** | 서버가 열려있는 "포트 번호". 같은 컴퓨터에서 여러 서버를 구분하는 번호 |
